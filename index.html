L'infrastructure physique d'Internet

Cours · Bloc 1 / 4

Tu imagines peut-être tes données voyager "dans le cloud". La réalité est bien plus spectaculaire — et physique.

99% du trafic = câbles sous-marins

Malgré le Wi-Fi et les satellites, 99% du trafic Internet mondial transite par des câbles posés au fond des océans. Ces câbles à fibre optique transportent les données sous forme d'impulsions lumineuses, à des vitesses proches de celle de la lumière.

🔍 Trois composants à ne pas confondre

ComposantRôleAnalogie
🔀 Routeur Reçoit les paquets de données et décide du chemin le plus rapide pour les acheminer vers leur destination, à travers le réseau mondial. Le panneau de signalisation sur l'autoroute — il oriente le trafic sans stocker quoi que ce soit.
🏭 Datacenter Bâtiment climatisé et sécurisé qui héberge des milliers de serveurs. C'est là que sont physiquement stockées les données des sites web, applications et services cloud. L'entrepôt — il stocke les marchandises (données) sans les livrer lui-même.
🖥️ Serveur Ordinateur puissant situé dans un datacenter qui répond aux requêtes des clients (navigateurs). Il stocke les pages web et les envoie quand on les demande. Le vendeur dans l'entrepôt — il reçoit ta commande et te livre la bonne page web.

✅ La souveraineté numérique — la capacité d'un État à maîtriser ses propres infrastructures numériques. Un débat politique très actuel en Europe.

Internet ≠ le Web

Cours · Bloc 2 / 4

C'est la confusion la plus répandue. Pourtant, la distinction est fondamentale.

🔌 Internet🕸️ Le Web
Réseau physique mondial d'appareils connectésUn service qui fonctionne sur Internet
Né dans les années 1960 (ARPANET)Inventé en 1989 par Tim Berners-Lee au CERN
Câbles, routeurs, datacentersPages consultées via un navigateur et des URL
💡 Analogie : Internet = le réseau routier. Le Web = les voitures. Mais d'autres "véhicules" utilisent aussi ce réseau : emails, appels vidéo, jeux en ligne, streaming musical… Aucun n'est "le Web".

✏️ 5 questions sur la vidéo, le cours et les protocoles
1. Dans la vidéo, quelle analogie est utilisée pour expliquer la relation entre Internet et le Web ?
Internet est une voiture, le Web est sa carrosserie
Internet est le Web en version améliorée
Internet est le réseau routier, le Web est un type de véhicule qui l'emprunte
Internet est un navigateur, le Web est le serveur
❌ Internet = les routes et autoroutes. Le Web = un type de véhicule parmi d'autres (emails, jeux en ligne sont d'autres véhicules). Cette analogie est au cœur de la vidéo.
✅ L'analogie de la route : Internet est l'infrastructure (les routes), le Web est un usage parmi d'autres, comme un type de véhicule parmi d'autres.
2. Quel protocole le Web utilise-t-il pour échanger des données entre un navigateur et un serveur ?
SMTP (Simple Mail Transfer Protocol)
HTTP / HTTPS (HyperText Transfer Protocol)
FTP (File Transfer Protocol)
TCP/IP (Transmission Control Protocol)
❌ Le Web repose sur HTTP (ou HTTPS pour la version sécurisée). SMTP est le protocole des emails, FTP sert au transfert de fichiers, TCP/IP est la base de tout Internet.
✅ HTTP/HTTPS — c'est le "langage" que parlent navigateur et serveur. Les autres protocoles (SMTP, FTP) existent aussi sur Internet, mais ne font pas partie du Web.
3. Lequel de ces usages utilise Internet SANS utiliser le Web ?
Consulter la page Wikipédia de "Tim Berners-Lee" dans Chrome
Envoyer un email depuis l'application Thunderbird (protocole SMTP)
Regarder une vidéo sur youtube.com dans Firefox
Lire un article sur lemonde.fr dans Safari
❌ Wikipedia, YouTube.com et lemonde.fr sont consultés via HTTP dans un navigateur — c'est du Web. Un client email comme Thunderbird utilise SMTP/IMAP, pas HTTP. Internet oui, Web non.
✅ Un client email utilise le protocole SMTP (pour envoyer) et IMAP (pour recevoir) — pas HTTP. Il utilise Internet sans passer par le Web.
4. TCP/IP est le protocole fondamental d'Internet. À quoi sert-il ?
Découper les données en paquets et les acheminer à travers les réseaux du monde entier
Chiffrer les données pour les sécuriser
Afficher les pages HTML dans un navigateur
Relier physiquement les câbles sous-marins entre eux
❌ TCP/IP est le protocole de base d'Internet (depuis 1983) : il découpe les données en petits paquets, les envoie par des chemins différents, et les recompose à l'arrivée.
✅ TCP/IP = le langage universel d'Internet. Il gère le découpage en paquets et leur acheminement. HTTP (le Web), SMTP (emails) et FTP (fichiers) fonctionnent tous par-dessus TCP/IP.
5. Qui a inventé le Web, et en quelle année ?
Tim Berners-Lee, en 1989, au CERN (Genève)
Bill Gates, en 1995, pour lancer Internet Explorer
Tim Berners-Lee, en 1969, pour relier les universités américaines
Marc Andreessen, en 1993, avec la création de Netscape
❌ Tim Berners-Lee, 1989, CERN. ARPANET date de 1969 (c'est Internet, pas le Web). Netscape (1994) est le premier navigateur grand public, pas l'inventeur du Web.
✅ Tim Berners-Lee, 1989, CERN. Il publie sa proposition d'un système de documents liés par hypertextes — ce qui deviendra le World Wide Web.

🌐 Et WWW, c'est quoi ?

💡 WWW ≠ Internet. Le World Wide Web est le nom de l'ensemble des pages web reliées par des liens hypertextes — c'est un service d'Internet, inventé par Tim Berners-Lee en 1989. Quand tu vois www. dans une URL, c'est simplement l'indication que tu accèdes au serveur web de ce domaine.
